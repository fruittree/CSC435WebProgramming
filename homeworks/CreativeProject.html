Overview

Throughout this course, we'll work with a number of different technologies to build different pieces of web applications. However, it can be de-motivating to build something to a specification, especially when you are learning something new and want to make something that has your own flavor. To give you a chance to play around with the technologies we're learning, this semester one of your homework assignments will be to write one or more websites from scratch, entirely of your own design. Every two weeks, there will be some requirements that you have to meet, but the requirements don't force you to have any particular content, layout, color scheme, feel, etc - you'll make that yourself.

The idea is that you spend around an hour each week. This should be ample time to meet the specified requirements for the week. If you find yourself spending much, much more than one hour per week, talk to your TA or the instructor, and we'll make sure that the time you spend in your creative project doesn't prevent you from getting other coursework done.

Each week you can decide if you want to build on one of your previous creative projects or start an entirely new one. Just make sure that you fulfill the requirements of the week (you don't need to meet the previous week's requirements).

Example Project Ideas

If you don't have inspiration for your own project, one suggestion is to build online experiment user interface to collect user responses. Such interface is critical for companies to gauge consumer interests. For example, you can show the user an image (e.g. a online fasion photo) and ask them to select a category of the garment by clicking on image icons of similar images with labels (e.g. wrap coat, A-line coat, Jacket, Woolcoat,Trench coat, etc). The experiment can have multiple trials. The website can have instructions and tutorials. Eventually, you can use online image database to generate the images and collect user responses in a efficient way. In addition, you can learn how to interact with server side scripts to save the data as well as visualize the data in real time. Finally, you can host the experiment on crowd-sourcing service such as Amazon Mechanical Turk. This is a very under explored field and might be very useful to share your final project on Github.

To get inspirations, you can explore Amazon Mechanical Turk's HITs. You can sign in as a worker and explore what are some of the crowd-sourcing experiments out there.

Another idea is to build a personal portofolio. This is a type of online showcase of your work. You can link into art projects, papers that you are proud of, etc. At the end of the quarter, you'll be able to share this portfolio with, well, anyone with an Internet connection :)
